A **commissioning protocol** defines the scope, procedures, roles, schedule, and acceptance criteria.

## Plan elements

**1. Scope:** Which systems will be commissioned? HVAC only, or also lighting, controls, DHW?

**2. Procedures:** How will each system be tested? (e.g., HVAC: measure static pressure, flow rate, setpoints; verify control sequences)

**3. Acceptance criteria:** What's the pass/fail threshold? (e.g., flow ±10% of design, temperature ±1°C setpoint, all sensors calibrated)

**4. Team roles:**
- **Commissioning agent:** Independent third party, leads testing
- **Contractor:** Provides equipment info, assists with adjustments
- **Designer:** Clarifies design intent if tests don't match
- **Owner/operator:** Observes, learns system, signs off

**5. Schedule:** How long will commissioning take? (2–8 weeks typical for new building, depending on size)

**6. Documentation:** What will be recorded? Test results, as-built drawings, manual settings, operator training.

## Common procedures

- **Flow balancing:** Measure and adjust flows at each terminal to match design
- **Sensor verification:** Check calibration, response time, location
- **Control sequence testing:** Step through each mode (heating, cooling, occupied, unoccupied) and verify correct response
- **Equipment performance:** Measure output (cooling capacity, heating capacity) vs. nameplate
- **Safety interlocks:** Verify emergency stops, failsafes work correctly

## Documentation package

After commissioning:
- Commissioning report (findings, defects, resolutions)
- As-built drawings (actual installed layout)
- Operation manuals (how to run each system)
- Maintenance schedule
- Operator training sign-off
- Warranty information

This package is handed over to the building owner/manager.
